## 4. 아키텍처 강점과 약점

### 💪 강점

1. **제로 의존성 배포** — Node/빌드 없이 정적 파일만으로 즉시 배포
2. **오프라인 동작** — localStorage 우선, Firebase는 선택적 동기화
3. **완성도 높은 디자인** — 다크 네이비 테마, CSS 변수 토큰화
4. **실시간 동기화** — Firestore `onSnapshot` + debounce
5. **게이미피케이션 완성도** — 직업·화폐·세금·쿠폰까지 현실 경제 시뮬레이션
6. **세이브/동기화 시스템** — SAVE_POINT.md 기반 PC/계정 간 작업 이어가기

### ⚠️ 개선 가능 영역

| 영역 | 현황 | 개선 방향 |
|------|------|-----------|
| **코드 구조** | `app.js` 14,301줄 단일 파일 | 기능별 모듈 분리 |
| **HTML 생성** | 문자열 연결로 DOM 생성 | 템플릿/컴포넌트화 |
| **상태 관리** | 매번 `getDb()` + `saveDb()` | 중앙 상태 + 이벤트 기반 |
| **반응형** | 칠판은 TV용, 기본은 반응형 | 모바일 학생 UI 최적화 |
| **테스트** | 없음 | 핵심 비즈니스 로직 단위 테스트 |
| **데이터 한도** | Firestore 문서 1MB | 대규모 시 분할 필요 |

## 5. 기술 상세

### 코드 스타일 규칙
- **JavaScript:** `var` 선언, ES5 호환, IIFE 패턴, `escapeHtml()` 필수
- **HTML:** 문자열 연결 (`+`) 방식 DOM 생성
- **CSS:** `:root` 변수 토큰 (다크 네이비 테마), BEM-like 명명, 고정 영역 `!important` 우선순위 제어
- **DB:** `getDb()` → 조작 → `saveDb(db)` 패턴
- **라우팅:** 해시 기반 (`#/teacher/...`, `#/student/...`)

### 핵심 알고리즘
- `boardDateKey()`: **오전 7시** 기준 하루 전환 (PRD 원본의 8시와 다름)
- `EXP_MAX = 100`: EXP 0~100% 스케일
- `applyDailyExpGrowthIfNeeded()`: 평일 +10%p 자동 성장
- `applyDailyBoardContentReset()`: 7시 경계 시간표·알림장 초기화
- `taxFromPayrollBase()`: 주급의 10% 세금 계산
- `adjustCorrespondingTaxForStudent()`: 주급 변경 시 동일 시점 국세청 연동 세금 정보 자동 보정
- `fiveWeekdayYmdsFromMonday()`: 1주(5일) 영업일 계산 알고리즘
- `resetTreasuryAndTaxHistory()`: 교사 암호 확인 후 세금 및 국고 완전 초기화
- `applyBulkStudentPatches()`: 엑셀을 이용한 일괄 수정 시, 순차 변경에 의한 일시적 정원 초과 오류 방지를 위해 `canAssignJob` 제한 조건 검사를 생략하고 강제 배정하도록 개선.
- `editDjRequest()`: 본인의 미승인 DJ 신청곡에 대해 제목을 수정할 때 `title`을 바꾸고 `createdAt`을 `Date.now()`로 새로고침하여 대기열 큐(오래된 순 정렬) 맨 뒤로 정렬이 변경되도록 함.
- `isUserTyping()`: `document.activeElement`를 검사하여 `textarea` 또는 특정 `input` 계열(텍스트 및 수치 입력형)에서 포커스가 활성화되어 사용자가 입력 중일 때 `true`를 리턴하며, 외부 실시간 동기화로 인한 `route()`(전체 UI 초기화)를 차단하여 입력 데이터 유실을 전방위 방지함.
- `submitDjRequest()`: DJ 신청 시 `boardDateKey()`를 기준으로 학급 전체의 오늘 등록 건수를 세어 8건 이상인 경우 신청을 마감 처리하여 등록을 차단함.
- `isSafeAvatarDataUrl()`: 아바타 이미지 유효성 검증 시 최대 150만 자의 base64 길이를 허용하고 이미지의 MIME 형태 검증 식을 유연하게 처리함.
- `buyStock() / sellStock()`: 1 Cal = 10,000원의 고정환율 기준, Cal 단위 또는 주(share) 단위 입력에 맞춰 소수점 4자리 한도의 fractional trading 거래를 처리하고 평단가/누적수량/잔고를 안전하게 갱신하는 거래 알고리즘.
- `fetchRealtimePrices()`: 교사가 지정한 GAS 프록시 주소로 비동기 GET 요청을 보내 등록된 종목들의 한국거래소 실시간 시세(원화 기준)를 한 번에 조회하고, `window.currentStockPrices` 전역 메모리에 실시간 캐싱하는 매커니즘.
- `renderStudentAssetChart()`: Chart.js 라이브러리를 동적 호출하여 학생의 보유 현금(Cal) 및 보유 주식 종목의 실시간 평가금액을 합산, 원형 Doughnut 차트로 시각화하며 Chart 인스턴스를 관리 및 폐기하는 렌더링 헬퍼.

### Firebase 구성
- **Auth:** 이메일/비밀번호 (선생님) + 익명 (학생)
- **Firestore:** `users/{uid}/classStatus/main` → `payloadJson` (전체 DB JSON)
- **Hosting:** 정적 파일, `firebase.json`에서 `public: "."`
- **SDK:** compat v10.7.1
