# Dev Bugfix Skill (버그 수정)

## 🎯 트리거 워드
- **"버그 수정"**, **"에러 수정"**, **"○○ 안 돼"**, **"○○ 고장났어"**

## 🛠️ 작동 절차

### 1단계: 증상 확인
- 어떤 모드? (선생님/학생), 어떤 화면? (해시 경로), 에러 메시지?
- Firebase 모드인지 로컬 모드인지

### 2단계: 코드 추적
- `app.js`에서 관련 함수를 **grep 검색**으로 탐색 (전체 읽기 금지)
- 데이터 흐름: `getDb()` → 조작 → `saveDb()` → `route()`
- CSS 레이아웃 문제면 `app.css` 해당 클래스 확인

### 3단계: 원인 분석 브리핑 출력
> **🐛 버그 분석:** 원인, 위치(함수명/줄), 영향 범위, 수정 방안

### 4단계: 최소 변경 원칙으로 수정
- 방어적 코딩: `typeof`, `null` 체크, 기본값 추가
- 기존 주석 유지, 사이드 이펙트 확인

### 5단계: 검증
- 선생님/학생 모드 양쪽 테스트
- Firebase 동기화 정상 확인

## 🔍 자주 발생하는 버그 패턴

| 패턴 | 원인 | 대응 |
|------|------|------|
| 학생 데이터 안 보임 | DB 동기화 지연 | `getDb()` null 체크 |
| 칠판 초기화 안 됨 | `boardDateKey()` 7시 경계 | 시간대 로직 확인 |
| Firebase 저장 실패 | 규칙 미배포 | `firestore.rules` 확인 |
| 삭제 후 잔여 데이터 | `removeStudentFromDb()` 누락 | 모든 배열 필터링 |
| 직업 배정 불가 | 한도 초과 | `canAssignJob()` 확인 |
