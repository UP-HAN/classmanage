# Dev Feature Skill (새 기능 개발)

이 스킬은 한의표 선생님이 학급운영도구에 새로운 기능을 요청할 때, **기존 코드 스타일·디자인 시스템·데이터 구조와 완벽하게 일관된 방식**으로 기능을 설계·구현·테스트하는 체계적인 개발 프로세스입니다.

## 🎯 트리거 워드 (Trigger Word)
- **"기능 추가"**, **"새 기능"**, **"○○ 만들어줘"** (구체적 기능 요청)

## 🛠️ 작동 절차 (Step-by-Step)

### 1단계: 요구사항 분석 및 영향 범위 파악
1. 선생님의 요구사항을 정리하여 **한 줄 기능 설명**으로 요약
2. 기존 코드에서 관련 부분을 검색:
   - `app.js`에서 관련 함수·뷰·라우트
   - `app.css`에서 관련 클래스
   - 데이터 모델(DB 스키마)에 영향이 있는지
3. **영향 범위 브리핑**을 선생님께 출력:

> **📋 기능 분석: {기능명}**
> - **수정 필요:** {파일 및 함수 목록}
> - **새로 추가:** {새 함수/클래스/데이터 필드}
> - **DB 변경:** {있으면 마이그레이션 전략}
> - **예상 난이도:** ⭐~⭐⭐⭐

### 2단계: 구현 (기존 패턴 준수)

#### 코드 스타일 규칙
- **JavaScript:** `var` 선언, 즉시 실행 함수 패턴 (`(function(){...})()`), ES5 호환
- **HTML 렌더링:** 문자열 연결 (`+`) 방식, `escapeHtml()` 필수 적용
- **이벤트:** `addEventListener` 직접 바인딩, 이벤트 위임 패턴
- **DB 접근:** `getDb()` → 수정 → `saveDb(db)` 패턴
- **라우팅:** 해시 기반 (`#/teacher/...`, `#/student/...`)

#### CSS 스타일 규칙
- **디자인 토큰:** `:root` CSS 변수 사용 필수 (`--bg-card`, `--accent`, `--gold` 등)
- **컴포넌트 명명:** BEM-like (`블록__요소--변형`)
- **버튼:** `btn btn--primary`, `btn--accent`, `btn--ghost`, `btn--danger`
- **패널:** `panel`, `panel__title`, `panel__text`
- **카드:** 둥근 모서리 (`var(--radius)`), 그림자 (`var(--shadow)`)

#### 데이터 모델 규칙
- 새 컬렉션 추가 시 `ensure{Collection}(db)` 함수 패턴 생성
- 학생 삭제 시 `removeStudentFromDb()`에 정리 로직 추가
- ID 생성: `C.uid()` 사용

### 3단계: 선생님/학생 모드 구분
- **선생님 전용:** `#/teacher/...` 경로에 라우트 추가
- **학생 전용:** `#/student` 내 분기 추가, 타 학생 데이터 접근 불가
- **직업 연동:** 특정 직업에 묶인 기능이면 `jobId` 체크 추가
- **내비게이션:** 기존 `app-nav`에 메뉴 항목 추가 (필요시 드롭다운)

### 4단계: Firebase 동기화 확인
- `saveDb(db)` 호출 시 자동으로 Firestore에 업로드됨 (debounce 2초)
- 새 데이터가 기존 `payloadJson`에 포함되는지 확인
- 1MB 한도 주의

### 5단계: 테스트 및 브라우저 확인
- 로컬에서 `firebase serve --only hosting` 또는 직접 HTML 열기
- 선생님 모드와 학생 모드 양쪽에서 테스트
- 디지털 칠판에 영향 없는지 확인
- 엑셀 내보내기/가져오기에 새 필드 반영 여부 확인

## ⚠️ 주의사항
- `app.js`가 이미 10,000줄이므로, 새 기능을 추가할 위치를 신중히 결정합니다.
- 관련 기능 근처에 함수를 배치하되, 나중에 모듈 분리가 용이하도록 의존성을 최소화합니다.
- `ensureDb()` 시드 데이터에 새 필드의 기본값을 추가해야 할 수 있습니다.
- **선생님 확인:** 학생에게 노출되는 기능은 반드시 선생님에게 확인 후 구현합니다.
