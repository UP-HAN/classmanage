# Dev Refactor Skill (코드 리팩토링)

## 🎯 트리거 워드
- **"리팩토링"**, **"코드 정리"**, **"모듈 분리"**, **"구조 개선"**

## 🛠️ 작동 절차

### 1단계: 현재 구조 분석
AI는 `app.js`의 기능 블록을 파악합니다:
- **유틸리티 함수** (escapeHtml, pad2, formatNum 등)
- **데이터 헬퍼** (getDb, saveDb, getStudent, activityLogsForStudent 등)
- **직업 시스템** (CLASS_JOBS, 은행/국세/통계/청소/쿠폰상인)
- **디지털 칠판** (boardDateKey, 날씨/급식 API, 시간표, 달력)
- **뷰/렌더링** (각 화면별 HTML 생성 함수)
- **라우팅** (해시 파서, route 분기)
- **인증** (Firebase Auth, 로그인 폼)
- **엑셀** (SheetJS 기반 가져오기/내보내기)

### 2단계: 리팩토링 계획 수립
- **안전 원칙:** 한 번에 하나의 모듈만 분리
- **테스트:** 각 분리 후 브라우저에서 전체 기능 확인
- **호환성:** `<script>` 태그 로딩 순서와 전역 변수 의존성 유지

### 3단계: 분리 실행 (권장 순서)

```
app/
├── js/
│   ├── utils.js          # 유틸리티 (escapeHtml, format 등)
│   ├── data.js           # DB 헬퍼, 데이터 조작
│   ├── jobs.js           # 직업 시스템 (CLASS_JOBS, 은행, 국세 등)
│   ├── digital-board.js  # 디지털 칠판 로직
│   ├── views/
│   │   ├── teacher.js    # 선생님 화면들
│   │   ├── student.js    # 학생 화면들
│   │   └── login.js      # 로그인 화면
│   ├── router.js         # 해시 라우팅
│   └── init.js           # 앱 초기화
├── app.css
└── index.html
```

### 4단계: CSS 정리 (선택적)
- 컴포넌트별 섹션 주석 정리
- 사용되지 않는 클래스 탐지 및 제거
- 미디어 쿼리 통합

### 5단계: 검증
- 모든 라우트 정상 동작 확인
- 선생님/학생 모드 전체 테스트
- Firebase 동기화 확인
- 디지털 칠판 + 7시 리셋 확인

## ⚠️ 주의사항
- `app.js`는 **즉시 실행 함수(IIFE)** 안에 모든 코드가 있으므로, 분리 시 **전역 네임스페이스 전략**을 세워야 합니다.
- 현재 zero-build 구조가 장점이므로, 빌드 도구 없이도 동작하도록 ES Modules 대신 전역 객체 패턴을 유지하는 것이 안전합니다.
- 리팩토링은 **선생님의 명시적 승인** 후에만 진행합니다 (기능 변경 없이 구조만 바꾸는 작업임을 설명).
