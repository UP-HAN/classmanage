# 🔄 Dev Sync Skill (동기화하기)

> **핵심 원리:** 프로젝트 폴더 안의 `[Han_Rule]/SAVE_POINT.md` 파일이 "기억 장치" 역할을 합니다.
> 어떤 PC, 어떤 계정, 어떤 AI 세션에서든 이 파일만 읽으면 이전 작업을 완벽히 이어갈 수 있습니다.

## 🎯 트리거 워드
- **"동기화해줘"**, **"동기화"**, **"싱크"**, **"이전 작업 이어서"**, **"어디까지 했지?"**

## 🛠️ AI가 수행할 절차

### 1단계: 프로젝트 기본 문서 읽기 (필수)
AI는 다음 파일들을 **정확히 이 순서대로** 읽어 프로젝트의 기본 맥락을 파악합니다:

1. **`[Han_Rule]/한의표 전용 안티그래비티 글로벌 룰.md`** — 작업 철학과 스킬 목록
2. **`[Han_Rule]/SAVE_POINT.md`** — ⭐ 이전 세션의 작업 상태 (이 파일이 핵심!)
3. **`[Han_Rule]/PROJECT_ANALYSIS.md`** — 📊 프로젝트 심층 분석 보고서 (구조·기능 맵·개선점·미답변 질문)
4. **`Pre-PRD.MD`** — 제품 요구사항 (구현 시 이 문서 우선)
5. **`프로그램초안설명서.MD`** — 기술 스택·디자인 토큰·파일 구조

### 2단계: SAVE_POINT.md 분석
세이브 파일에서 다음을 추출합니다:

- **마지막 세이브 시각** → 얼마나 오래 전인지
- **완료된 작업** → 이미 끝난 일이 무엇인지
- **변경된 파일 목록** → 어떤 파일이 최근에 수정되었는지
- **변경 상세** → 코드 맥락과 변경 이유
- **미완성 작업** → 지금 바로 이어서 해야 할 일
- **알려진 이슈** → 주의해야 할 버그
- **다음 AI에게 전하는 메모** → 특수한 맥락과 주의점

### 3단계: PROJECT_ANALYSIS.md에서 전체 맥락 파악
분석 보고서에서 다음을 확인합니다:
- **현재 구현된 기능 전체 목록** → 프로젝트가 어디까지 완성되었는지
- **아키텍처 강점과 약점** → 코드 작업 시 주의할 점
- **미답변 질문** → 선생님께 아직 확인이 필요한 사항
- **개선 가능 영역** → 리팩토링·기능 추가 후보

### 4단계: 변경된 파일 현재 상태 확인
SAVE_POINT.md에 기록된 변경 파일들의 **현재 상태**를 실제로 확인합니다:
- 기록된 변경 사항이 실제 코드에 반영되어 있는지
- 세이브 이후 누군가 추가 수정한 부분은 없는지
- 파일이 누락되거나 깨진 것은 없는지

### 5단계: 동기화 완료 브리핑 출력

```
🔄 동기화 완료 — 학급운영도구 프로젝트

📅 마지막 세이브: {날짜/시간}
✅ 이전에 완료한 작업:
   - {작업 1}
   - {작업 2}

⏳ 이어서 해야 할 작업:
   - {미완성 작업 1}
   - {미완성 작업 2}

🐛 주의할 이슈:
   - {이슈 1}

💡 맥락 메모:
   - {이전 AI가 남긴 메모}

👉 바로 이어서 작업할 수 있습니다. 무엇부터 할까요?
```

### 6단계: 미완성 작업이 있으면 즉시 제안
미완성 작업 목록이 있다면, 가장 우선순위가 높은 것부터 시작하도록 제안합니다.

## 📌 SAVE_POINT.md가 없는 경우
세이브 파일이 없다면(첫 세션이거나 실수로 삭제된 경우):
1. 선생님께 알림: "세이브 포인트가 없습니다. 프로젝트를 처음부터 스캔합니다."
2. 프로젝트 전체 구조를 스캔하여 현재 상태를 파악
3. 핵심 파일들을 읽어 프로젝트 이해도를 확보
4. 브리핑 출력 후 선생님의 지시를 기다림

## ⚠️ 주의사항
- `app.js`는 10,000줄 이상이므로 **전체를 읽지 않습니다.** SAVE_POINT.md에 기록된 변경 부분만 확인합니다.
- 동기화 후 바로 코드를 수정하지 않습니다 — 항상 선생님의 지시를 먼저 기다립니다.
- 세이브 파일의 내용과 실제 코드가 다르면, 실제 코드를 우선으로 삼고 선생님께 보고합니다.
