# 프로그램 초안 설명서 — 학급 RPG · STATUS

**목적:** `Pre-PRD.MD`와 함께 사용하여 **동일 UI/UX·동일 동작**으로 재구현하기 위한 기술·디자인 명세이다.  
**스택:** 순수 HTML/CSS/바닐라 JS, **의존성:** `js/core.js`(저장·세션·해시), `app/app.js`(라우팅·뷰), `app/app.css`, SheetJS `xlsx` CDN.

---

## 1. 파일 구조

```
학급운영도구/
  index.html              # ensureDb 후 app/index.html 로 replace 리다이렉트
  js/
    core.js               # ClassStatusCore: loadDb, saveDb, ensureDb, session, uid, hashPassword
  app/
    index.html            # #app 루트, core.js + xlsx + app.js
    app.js                # 전체 라우팅·화면·디지털 칠판 바인딩
    app.css               # 전역 + 디지털 칠판 TV 스타일
  teacher/
    index.html            # 선택: #/teacher 로 진입용
```

## 2. 저장소·세션

| 키 | 용도 |
|----|------|
| `localStorage` `class-status-db-v1` | 전체 DB JSON |
| `sessionStorage` `classStatusSession` | 로그인 세션 JSON |

**ensureDb():** DB 없거나 users 비면 시드 생성 — 선생님 1명 `loginId: "teacher"`, 비번 `demo123` (SHA-256 + salt).

**getDb():** 매번 `JSON.parse(localStorage)` — 참조가 바뀔 수 있으니 저장 후 동기화 주의.

---

## 3. 해시 라우팅

- `parseHash()`: `#/a/b` → `["a","b"]`  
- **선생:** `parts[0]==="teacher"` (없으면 `#/teacher`로)  
- **학생:** `parts[0]==="student"` 아니면 `#/student`로  

`init()`에서 `ensureDb().then` 후 `requireSession()` → 기본 해시 설정 → `route()` + `hashchange`.

## 4. 핵심 알고리즘

### 4.1 `boardDateKey()` (활동·일일 콘텐츠 공통)

- 현재 시각 기준 **오전 8시 미만이면 전날 날짜**를 캘린더 일로 본다.  
- 반환: `YYYY-MM-DD` 문자열.

### 4.2 학생 활동 (`boardActivity`)

- 구조: `digitalBoard.boardActivity[boardDateKey][studentId] = 숫자`  
- 8시 넘어가면 `boardDateKey`가 바뀌어 **자동으로 새 날짜 버킷**(기본 0).

### 4.3 일일 콘텐츠 리셋 (`applyDailyBoardContentReset`)

- 필드 `digitalBoard.dailyContentKey`  
- **최초 없음:** 현재 `boardDateKey`만 저장, **내용 삭제 안 함** (기존 사용자 마이그레이션).  
- **저장된 키 ≠ 현재 키:**  
  - 시간표 `morning`, `p1`…`p6` 전부 `{ subject: "", activity: "" }`  
  - `notice` 빈 문자열  
  - `dailyContentKey` 갱신  
- **유지:** `mealApi`, `weather`, `todos`, `calendarMemos`, `noticeFontPx`, `boardActivity` 구조 자체.

### 4.4 학생 삭제 `removeStudentFromDb`

- `students`, `users`(studentId 일치), `titleGrants`, `activityLogs`, `behaviorNotes` 필터  
- `boardActivity`의 **모든 날짜 객체**에서 해당 `studentId` 키 삭제

## 7. 외부 API

| 용도 | URL 패턴 |
|------|-----------|
| 날씨 | `https://api.open-meteo.com/v1/forecast` — current temp, weather_code, hourly pop |
| 미세 | `https://air-quality-api.open-meteo.com/v1/air-quality` — pm2_5 |
| 급식 | `https://open.neis.go.kr/hub/mealServiceDietInfo` — KEY, ATPT, SD_SCHUL_CODE, MLSV_YMD (YYYYMMDD) |

날씨/급식은 브라우저 CORS에 따라 실패 메시지 표시 가능.

## 14. 구현 시 주의

1. **HTML 이스케이프:** 사용자 입력·DB 문자열은 `escapeHtml` 후 삽입.  
2. **디지털 칠판 언마운트:** `window.__digitalBoardCleanup`으로 인터벌·키 리스너 제거.  
3. **로그인 UI 제거된 상태**가 현재 기준이므로, 재구현 시에도 **선생 자동 세션**을 맞출지·진짜 로그인을 넣을지 Pre-PRD와 일치시킬 것.
